# Candidate Profile & AI Autofill Workflow

_Last updated: 2026-05-04_

---

## Overview

Parvagas allows candidates to build their professional profile incrementally — no CV required. Every section can be saved partially. When a CV is uploaded, the AI service extracts data and proposes it for the candidate to review and confirm before any profile field is changed.

---

## 1. Onboarding Flow (First Login)

On the candidate's **first visit** to `/Portal/Candidato`, the system checks `localStorage` for the key `parvagas_onboarding_done`:

| Condition | Behaviour |
|-----------|-----------|
| Key absent | Redirect → `/Portal/Candidato/Onboarding` |
| Key present | Redirect → `/Portal/Candidato/Meu-Perfil` |

### Onboarding Page (`/Portal/Candidato/Onboarding`)

Presents three choices:

1. **Carregar o meu CV** — navigates to `/Portal/Candidato/CV-e-Documentos`.
2. **Descarregar modelo de CV** — downloads `/templates/modelo-cv-parvagas.docx` then navigates to CV upload after 800ms.
3. **Pular e preencher manualmente** — navigates directly to `/Portal/Candidato/Meu-Perfil`.

All three choices write `parvagas_onboarding_done=1` to `localStorage` before navigating, so the prompt is never shown again.

## 2. CV Template

File: `public/templates/modelo-cv-parvagas.docx`

Generated by `server/scripts/generateCvTemplate.mjs` (zero npm dependencies — uses only Node.js builtins to build a valid DOCX/ZIP).

The template contains sections for:
- Dados Pessoais
- Resumo Profissional
- Experiência Profissional
- Educação
- Competências
- Idiomas
- Certificações
- Preferências de Emprego

Regenerate the file any time with:
```bash
node server/scripts/generateCvTemplate.mjs
```

## 3. Partial Profile Saves

### Backend

- Endpoint: `PATCH /candidates/profile`
- Handler: `updateMyProfile` in `server/controller/candidates.js`
- Strategy: **read → merge → soft-validate → persist**

```
1. Load existing CandidateProfile for the user (or empty object if none).
2. Merge incoming request body on top of existing data.
3. Normalize the merged payload (trim strings, lowercase email, deduplicate arrays, etc.).
4. Run format-only validation (validateProfileFormatOnly):
   - Only fails on *format errors* (invalid email regex, invalid enum value if provided, negative salary).
   - Never fails on empty / missing required fields.
5. Persist via findOneAndUpdate with upsert: true.
6. Return { profile, completionScore, isPartial }.
```

`isPartial: true` is returned when one or more key fields (fullName, email, phone, location, professionalTitle, summary, preferredJobType, availability, skills) are empty.

### Frontend

The profile form has two save actions:

| Button | Behaviour |
|--------|-----------|
| **Guardar progresso** | Saves and stays on the page; shows success toast. |
| **Guardar e continuar depois** | Saves, shows toast, then redirects to `/Portal/Candidato/Dashboard`. |

Both buttons call the same `PATCH /candidates/profile` endpoint. Validation only blocks on format errors — missing fields never prevent saving.

## 4. Profile Completion Score

`calculateProfileCompletion(profile)` in `server/services/matchingService.js` checks:

| Field | Weight |
|-------|--------|
| fullName | 1/12 |
| email | 1/12 |
| phone | 1/12 |
| location | 1/12 |
| professionalTitle | 1/12 |
| summary | 1/12 |
| skills (≥ 1) | 1/12 |
| experience (≥ 1) | 1/12 |
| education (≥ 1) | 1/12 |
| preferredJobType | 1/12 |
| availability | 1/12 |
| expectedSalaryAoa | 1/12 |

The percentage (0–100) is stored in the `completionScore` field of `candidate_profiles` payload and returned with every profile response.

The **candidate portal sidebar** fetches `completionScore` on mount and shows an amber pulsing dot on the "Meu Perfil" nav item when the score is below 100%.

## 5. CV Upload & AI Parsing

### Upload flow

1. Candidate visits `/Portal/Candidato/CV-e-Documentos` and drops a file (PDF / DOC / DOCX, max 8 MB).
2. Frontend posts to `POST /candidates/cv/parse` with `multipart/form-data`.
3. Backend:
   a. Validates file type and size.
   b. Uploads to Supabase Storage via `storageService`.
   c. Creates a `CandidateDocument` record.
   d. Creates an `AIParseRun` record (status: started).
   e. Extracts text (`cvTextExtractorService` — mammoth for DOCX, pdf-parse for PDF).
   f. Calls `parseCvToProfile(text)` in `aiService` (real LLM or fallback regex parser).
   g. Updates `AIParseRun` to status: success.
   h. Returns `{ parseRunId, documentId, profileDraft, missingFields, requiresCandidateApproval: true }`.

4. Frontend stores `profileDraft` in `sessionStorage` under key `parvagas_cv_parse_draft`.
5. Frontend shows the extracted data in an editable inline review form.

### AI Suggestion Banner (Meu-Perfil)

When the candidate navigates to `/Portal/Candidato/Meu-Perfil`, the page reads `sessionStorage` for `parvagas_cv_parse_draft`. If a draft is found and not yet dismissed, an info banner appears:

> **"Sugestões do CV prontas para aplicar"**  
> _O seu CV foi processado com IA. Clique em "Aplicar sugestões" para preencher automaticamente os campos em branco — pode editar tudo antes de guardar._

Clicking **"Aplicar sugestões"** calls `applyAiDraft()` which:
- **Never overwrites** existing user data.
- Only fills in fields that are currently empty.
- Removes the sessionStorage entry after applying.
- Fires a success toast: "Sugestões do CV aplicadas. Revise e guarde o perfil."

Clicking **✕** dismisses the banner without applying anything.

### Approve & Save (in CV-e-Documentos)

After reviewing the extracted fields in the inline form, clicking **"Confirmar e guardar no perfil"** calls `POST /candidates/profile/approve`:

```json
{
  "profileDraft": { /* edited extracted fields */ },
  "parseRunId": "...",
  "consentGiven": true
}
```

The `approveAndSaveProfile` controller runs full `validateProfilePayload` before persisting. If the draft is incomplete, the candidate can still save the partial profile separately via `PATCH /candidates/profile`.

## 6. AI Professional Summary

Endpoint: `POST /candidates/profile/summary-draft`

The candidate can click **"Melhorar resumo com IA"** in the Resumo section. The backend calls `generateProfessionalSummaryDraft(profile)` in `aiService.js`, which constructs a structured prompt from the candidate's existing data and returns `{ draft, warning }`.

The generated draft is shown in an amber review card. The candidate can:
- Click **"Aplicar resumo"** to copy the draft into the summary textarea.
- Click **"Regerar"** to request a new draft.
- Ignore the suggestion entirely.

## 7. Navigation

### "Continue Profile" sidebar badge

`CandidateSidebar` fetches `completionScore` on mount and renders an amber pulsing dot on the **Meu Perfil** nav item when the score is below 100%.

### "Ir para secção incompleta" button

A tertiary button in the Meu-Perfil form footer (visible only when `completion < 100%`) calls `goToFirstIncomplete()`, which:
1. Checks sections in order: informacao → experiencia → educacao → skills → resumo.
2. Finds the first section not in `"complete"` state.
3. Sets it as the open section and scrolls to it smoothly.

## 9. Integration Test Coverage

`server/tests/integration.test.js` covers:

- Partial profile save persists without all required fields → HTTP 200, `isPartial: true`
- Incremental PATCH merges correctly (existing fields preserved)
- Invalid email format returns 400 with `fieldErrors`
- Full profile draft approval → HTTP 200
- `POST /candidates/profile/summary-draft` returns `draft` or `warning`

Run tests (requires `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY`):
```bash
NODE_ENV=test node --test server/tests/integration.test.js
```
